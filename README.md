Первый запуск приложения происходит через:

```make run-docker-build```

То же самое что и ```docker-compose up --build```

Для последующих запусков можно использовать:

```make run-docker```


Протестить api можно через swagger:

<localhost:8081>


При локальном запуске через команду ```make run-local```, создать файл ```.env``` по примеру из ```.env_example```.


Примечания к реализации:

1. Все передаваемые айди (например, user_id) будут являться типом uint, будут игнорированы и сгенерированы автоматически БД primary key
2. В слое данных может содержаться бизнес логика: сделано для уменьшения кода. Например в методе ReassignReviewer содержится вся логика.


Доп задания:
1. Реализована ручка /users/getPR для получения всех PR пользователя с данным статусом. Входные параметры: параметры запроса user_id и stats.
2. Выполнено нагрузочное тестирование с помощью утилиты [bombardier](https://github.com/codesenberg/bombardier). Протестировал самую сложную ручку: /pullRequest/reassing.
Были добавлены в БД 10 активных разработчиков с именами Razrab1, Razrab2, ..., Razrab10. Создан пул руквест, где Razrab1 автор. (Вообще проводить нагрузочное тестирование не стоит делать на той же машине, где запущен сервер, но у меня нет денег на свой сервер).

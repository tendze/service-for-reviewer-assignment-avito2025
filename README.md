### Первый запуск приложения происходит через:
```make run-docker-build```

То же самое что и ```docker-compose up --build```

Для последующих запусков можно использовать:

```make run-docker```

### Swagger

### Локальный запуск
При локальном запуске через команду ```make run-local```, нужно создать файл ```.env``` по примеру из ```.env.example```. (нужно просто указать одну переменную - ```DB_PASSWORD```). Затем поднимите БД вручную либо через докер с данными из ```./config/local.yaml```


### Примечания к реализации:

1. Все передаваемые айди (например, user_id) будут являться типом uint, будут игнорированы и сгенерированы автоматически БД primary key
2. В слое данных может содержаться бизнес логика: сделано для уменьшения кода. Например в методе ReassignReviewer содержится вся логика ручки ```/pullRequest/reassign```.
3. В файле ```docker-compose.yaml``` указан пароль в открытом виде для удобства запуска. В идеале хранить пароль в гитхаб сикретс либо в отдельных сервисах как вольт.
